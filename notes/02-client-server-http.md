# Client, Server and HTTP

## Client

A client is any application that sends requests.

Examples:

* Browser
* Mobile App
* React Frontend
* Postman

## Server

A server receives requests, processes them, and sends responses.

Examples:

* FastAPI
* Django
* Flask

## HTTP

HTTP is the communication protocol between client and server.

Request → Server → Response

## Common HTTP Methods

GET - Retrieve data

POST - Create data

PUT - Replace existing data

PATCH - Update specific fields

DELETE - Remove data

## Status Codes

200 - Success

201 - Resource Created

400 - Bad Request

401 - Unauthorized

404 - Not Found

500 - Internal Server Error

## Key Learning

Every backend application works around requests, processing, and responses.
